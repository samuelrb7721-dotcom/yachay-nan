# Yachay Ñan - Development Rules

## Project

Educational Flutter application about the history, culture, geography and gastronomy of Peru.

## Tech Stack

- Flutter
- Dart
- Material 3

## Git Workflow

Never work directly on main.

Workflow:

main
↓
develop
↓
feature branches

Example:

feature/login

feature/map

feature/gastronomy

feature/profile

## Coding Rules

- Use clean architecture.
- One widget per file.
- Avoid duplicated code.
- Reusable widgets.
- Keep business logic outside UI.
- Use English for code.
- Use Spanish for application texts.

## Before every change

Read this file.

Explain the plan.

Modify only necessary files.

Do not delete existing code.

Run flutter analyze before finishing.
